CSharp提示本

发表于2023-09-26|更新于2024-06-25|编程

Csharp编写提醒

命名必须看得懂哪怕很长

不要用单字符命名，但如for循环一样的条件句除外

不要在方法中写不明所以的数字，一开始就应该进行明确的声明

不要手写字符串，而是生成字符串，避免字符串误写

不要总使用 Public ,要考虑 private 等
越少访问意味着更好的管理

避免使用 Tag 和 String，代码多时维护起来不堪重负；
但在单例模式等中可以使用

应多思考高内聚低耦合，进行代码解耦，不能出现相互调用类，考虑事件

面向接口设计可以更好的解耦，进行项目拆分，上层项目不需要考虑下层项目的调用，下层项目提供接口给上层项目实现即可

Csharp命名规范

其他

[SerializeField]：公开但不能外部访问

尽量使用 TextMeshPro using TMpro

涉及到UI方面：尽可能用事件去“通知”而不是杂糅在一起；

使用 输入系统 “Input System” 而不是 输入管理器 “Input Manager”

推荐使用 “URP” 更适合各个平台
